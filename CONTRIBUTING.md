# Contributing to DSA Master Hub

Thanks for your interest in contributing.

## How to Contribute
1. Fork the repository.
2. Create a new branch for your change.
3. Make focused, topic-specific changes.
4. Test your changes locally.
5. Open a clear pull request.

## Branch Naming
Use descriptive names, for example:
- `feature/arrays-two-sum`
- `fix/input-validation-arrays`
- `docs/readme-improvements`

## Code Guidelines
- Keep code simple and beginner-friendly.
- Prefer clear variable names and readable logic.
- Add input validation and basic exception handling where needed.
- Keep templates, styles, and scripts organized.

## Commit Message Style
Use short, clear messages, for example:
- `feat: add linked list insert operation page`
- `fix: handle empty array input in arrays view`
- `docs: update setup instructions`

## Pull Request Checklist
- Code runs without errors.
- Change is limited to the intended topic.
- README/docs updated if behavior changed.
- No unrelated files modified.

## Reporting Issues
When reporting a bug, include:
- What you expected
- What happened
- Steps to reproduce
- Environment details (OS, Python, Django versions)
